# 数据库索引优化

本文介绍数据库索引的优化策略。

## 索引类型

### B-Tree 索引
最常见的索引类型，适用于范围查询。

### Hash 索引
适用于精确匹配查询。

### 全文索引
适用于文本搜索。

## 优化技巧

### 1. 选择合适的列

```sql
-- 为经常查询的列创建索引
CREATE INDEX idx_user_email ON users(email);

-- 复合索引，注意列的顺序
CREATE INDEX idx_order_user_date ON orders(user_id, created_at);
```

### 2. 避免过多索引

索引会增加写入开销。

### 3. 使用覆盖索引

避免回表查询。

## 监控和诊断

- 使用 EXPLAIN 分析查询
- 监控慢查询日志
- 定期维护索引

## 总结

合理的索引设计可以显著提升查询性能。
